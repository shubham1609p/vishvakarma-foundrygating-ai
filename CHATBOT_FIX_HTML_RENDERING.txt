                        CHATBOT HTML RENDERING FIX
═══════════════════════════════════════════════════════════════════════════════

🐛 ISSUE FOUND:
Raw HTML tags (<strong>, </strong>, etc.) were showing as text in chat messages
instead of being rendered as formatted text.

Example of what was happening:
  "Defects analysis for <strong>Bevel Gear</strong>"
  
Should show as:
  "Defects analysis for Bevel Gear" (with "Bevel Gear" in bold)

═══════════════════════════════════════════════════════════════════════════════

ROOT CAUSE:
The addChatMsg() function was ALWAYS sanitizing HTML, even when we intentionally
wanted to pass formatted HTML for user messages and AI responses.

The sanitizeHTML() function uses textContent, which escapes all HTML:
  <strong> → &lt;strong&gt;
  </strong> → &lt;/strong&gt;

This is good for security (prevents XSS), but bad when we have legitimate HTML
we want to display.

═══════════════════════════════════════════════════════════════════════════════

✅ FIX APPLIED:

1. MODIFIED addChatMsg() function:
   ─────────────────────────────
   Added a third parameter "isHTML" (optional):
   
   OLD: addChatMsg(html, type)
   NEW: addChatMsg(text, type, isHTML)
   
   If isHTML=true: Use innerHTML directly (for pre-formatted HTML)
   If isHTML=false/undefined: Sanitize first (for user input text)

2. UPDATED formatAI() function:
   ──────────────────────────
   Wraps error text with sanitizeHTML() to prevent any issues:
   
   OLD: '<span style="color:var(--ember);">' + text + '</span>'
   NEW: '<span style="color:var(--ember);">' + sanitizeHTML(text) + '</span>'

3. UPDATED sendChatMsg() function:
   ────────────────────────────
   Now calls: addChatMsg(formatAI(reply), 'ai', true)
   
   The formatAI() returns safe HTML, so we pass isHTML=true
   to avoid double-sanitization.

4. UPDATED quickAsk() function:
   ──────────────────────────
   Now calls: addChatMsg(..., 'user', true)
   for the quick ask title message
   
   And: addChatMsg(formatAI(reply), 'ai', true)
   for AI responses.
